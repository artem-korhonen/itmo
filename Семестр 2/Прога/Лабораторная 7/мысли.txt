transport-request теперь имеет окно действия (строка, в которой отображается, что будет происходить: autorize, login, command)

autorize
передаётся информация о человеке (ФИО, дата рождения, юзернейм и пароль)
сервер принимает информацию, обратно возращает True, если юзернейм уникальный, иначе False

login
передаётся юзернейм и пароль
сервер проверяет соответствие, возращает True при удачном входе, иначе False

command 
передаются данные для команды
сервер возращает результат команды

проверка на авторизацию для команды будет на клиенте
