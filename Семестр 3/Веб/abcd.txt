преобразование html страницы до полностью зарендеренного dom + css дерева, алгоритмы графов на dom дереве, как найти по id, по классу

1) HTML -> DOM
html читается сверху вниз
первый тег <html> - корневой элемент дерева
дочерние узлы - <head>, <body>
парсер добавляет узлы по мере чтения
тексты, атрибуты - тоже узлы
когда файл доходит до конца файла - DOM построен
2) CSS -> CSSOM
браузер находит стили, загружает их
парсер css разбивает текст на токены
из токенов строится дерево CSSOM
объединение всех источников стилей
3) DOM + CSSOM -> RenderTree
Браузер объединяет деревья
Каждый видимый элемент из DOM получает свои стили из CSSOM
Невидимые элементы не попадают в RenderTree

синхронная и асинхр обработка (как работает?)

1) Синхронная
функции выполняются последовательно, по шагам: пока один запрос не завершится, следующий не начнётся
если синхронный вызов занимает много времени, браузер может заблокировать страницу
1 поток = 1 запрос
2) Асинхронная
запрос отправляется, но программа не ждёт его завершения
когда ответ приходит, вызывается функция, обрабатывающая его
1 поток = много запросов для тысяч людей

идемпонентность http методов
get - yes
post - no
put - yes
head - yes

put vs patch vs post

patch - метод для частичного обновления ресурса (частичные обновления, оптимизация ресурсов, параллельные обновления) (может быть идемпонентен, может - нет)
put - метод для полного обновления ресурса (обычно отправляется полное тело ресурса) (идемпонентен)
post - метод для создания ресурса (не идемпонентен - при отправке запроса повторно создаются разные объекты)

preflight запрос
options запрос, который отправляется браузером перед основным запросом к серверу. Это часть механизма безопасности, связанного с CORS в веб-разработке

access-control-allow-origin - указывает, какие домены могут получать ответ
